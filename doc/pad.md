# demo

这份文件只记录 4 类内容：

1. 开锁命令
2. 开锁反馈
3. 手动关锁反馈
4. 电磁锁长通电 / 关闭长通电

## 1. 基本说明

- 通讯方式：`RS-485`
- 串口参数：`9600, 8N1`
- 数据格式：十六进制 `HEX`
- 校验方式：`BCC = 前面所有字节逐个异或(XOR)`

示例：

```text
8A 01 01 11 -> BCC = 8A ^ 01 ^ 01 ^ 11 = 9B
```

完整报文：

```text
8A 01 01 11 9B
```

## 2. 开锁命令

### 2.1 报文格式

```text
8A [板地址] [锁地址] 11 [BCC]
```

字段说明：

- `8A`：开锁命令头
- `板地址`：锁控板地址，来自板上的地址设置
- `锁地址`：要打开的锁编号
- `11`：开锁功能码，固定值
- `BCC`：异或校验

### 2.2 示例

打开 `1` 号板的 `1` 号锁：

```text
8A 01 01 11 9B
```

打开 `1` 号板的 `2` 号锁：

```text
8A 01 02 11 98
```

## 3. 开锁反馈

### 3.1 报文格式

```text
8A [板地址] [锁地址] [状态] [BCC]
```

固定长度：`5` 字节

字段说明：

- `8A`：反馈命令头
- `板地址`：当前锁控板地址
- `锁地址`：当前锁编号
- `状态`：开锁后的反馈状态
- `BCC`：异或校验

### 3.2 状态含义

这里最关键的是：`状态` 的含义和锁的反馈类型有关。

#### 开门反馈型锁

- `11`：开锁成功 / 当前为开
- `00`：开锁失败 / 当前未开

#### 关门反馈型锁

- `00`：开锁成功 / 当前为开
- `11`：开锁失败 / 当前为关

### 3.3 示例

开锁成功示例，常见于开门反馈型锁：

```text
8A 01 01 11 9B
```

开锁失败示例，常见于开门反馈型锁：

```text
8A 01 01 00 8A
```

说明：

- 同样是 `11 / 00`
- 一定要结合锁是“开门反馈”还是“关门反馈”来解释

## 4. 手动关锁反馈

### 4.1 先说明一件事

普通电控锁协议里**没有单独的“关锁命令”**。

也就是说：

- 你发命令时，只有“开锁命令”
- 门被人手动关上后，板子反馈的是“状态变化”
- 不是“执行关锁成功应答”

### 4.2 主动上报格式

如果设备支持主动上报，手动关门后会主动发送：

```text
81 [板地址] [锁地址] [锁状态] [BCC]
```

固定长度：`5` 字节

字段说明：

- `81`：状态变化主动上报
- `板地址`：当前锁控板地址
- `锁地址`：当前锁编号
- `锁状态`：门被手动关上后的当前状态
- `BCC`：异或校验

### 4.3 手动关门后的状态含义

#### 开门反馈型锁

- 手动关门后常见上报为 `00`

#### 关门反馈型锁

- 手动关门后常见上报为 `11`

### 4.4 示例

PDF 给出的手动关门主动上报示例：

```text
81 01 02 00 82
```

这条报文表示：

- `01` 号板
- `02` 号锁
- 当前状态位是 `00`

如果该锁是开门反馈型锁，那么这通常表示：

- 门已经被手动关上
- 反馈触点已经回到“关门状态”

### 4.5 你实测到的微动反馈

你实测到的 `02` 号锁微动反馈如下：

#### 微动按下反馈

```text
81 01 02 11 93
```

格式拆解：

```text
81 [板地址] [锁地址] [状态] [BCC]
81  01      02      11    93
```

状态说明：

- `81`：状态变化主动上报
- `01`：1 号板
- `02`：2 号锁
- `11`：微动按下
- `93`：异或校验

校验计算：

```text
81 ^ 01 ^ 02 ^ 11 = 93
```

#### 微动松开反馈

```text
81 01 02 00 82
```

格式拆解：

```text
81 [板地址] [锁地址] [状态] [BCC]
81  01      02      00    82
```

状态说明：

- `81`：状态变化主动上报
- `01`：1 号板
- `02`：2 号锁
- `00`：微动松开
- `82`：异或校验

校验计算：

```text
81 ^ 01 ^ 02 ^ 00 = 82
```

按你这次测试结果，可以直接记成：

- `11 = 微动按下`
- `00 = 微动松开`

如果后续你接的是同型号、同接法的锁，这组状态定义可以直接沿用。

## 5. 电磁锁长通电 / 关闭长通电

### 5.1 功能说明

这部分只适用于电磁锁。

PDF 原文含义是：

- 长通电：持续给锁通电，产生磁力，保持锁门
- 关闭长通电：停止持续通电，磁力消失，解锁
- 这个功能 `个别产品不带`

### 5.2 开启长通电触发指令

报文格式：

```text
9A [板地址] [锁地址] 11 [BCC]
```

字段说明：

- `9A`：开启长通电命令头
- `板地址`：锁控板地址
- `锁地址`：目标锁编号
- `11`：功能码，固定值
- `BCC`：异或校验

示例：

```text
9A 01 02 11 88
```

表示：

- 对 `02` 号板
- `01` 号锁
- 开启长通电

### 5.3 关闭长通电触发指令

报文格式：

```text
9B [板地址] [锁地址] 11 [BCC]
```

字段说明：

- `9B`：关闭长通电命令头
- `板地址`：锁控板地址
- `锁地址`：目标锁编号
- `11`：功能码，固定值
- `BCC`：异或校验

示例：

```text
9B 01 02 11 89
```

表示：

- 对 `02` 号板
- `01` 号锁
- 关闭长通电

### 5.4 长通电反馈

PDF 对这两条命令写的是：

- 发送 `9A` 后：`并反馈回传该通道的反馈状态`
- 发送 `9B` 后：`并反馈上传该通道当前状态`

但是原 PDF **没有单独列出 `9A/9B` 的专用反馈帧格式**。

所以这部分目前能明确的只有两点：

1. 执行 `9A/9B` 后，设备会回传该通道的当前反馈状态
2. 这个状态的解释规则，仍然按锁的反馈类型来理解

长度说明：

- 从 PDF 文字本身，**不能严格确认** `9A/9B` 的反馈是否固定为某个长度
- 如果设备复用了单锁状态反馈风格，那大概率也是 `5` 字节
- 但这一点需要以你的实机抓包结果为准

### 5.5 长通电反馈里的状态怎么理解

如果设备回传的是该通道当前状态，那么状态位仍按下面解释：

#### 开门反馈型锁

- `11`：开
- `00`：关

#### 关门反馈型锁

- `00`：开
- `11`：关

注意：

- 这个状态值表示的是反馈触点对应的当前状态
- 不要把它简单理解成 `11=已通电` 或 `00=已断电`
- 对电磁锁来说，是否正在长通电，和门/锁反馈状态不是完全同一个概念

### 5.6 联调建议

因为 PDF 没单独给 `9A/9B` 的反馈帧格式，实际联调建议这样做：

1. 先发送 `9A` 或 `9B`
2. 再发送单锁状态查询：

```text
80 [板地址] [锁地址] 33 [BCC]
```

这样最容易确认长通电开启或关闭后，该通道最终处于什么状态。

## 6. 联调时怎么理解状态

可以直接按下面理解：

### 开门反馈型锁

- 开锁后收到 `11`：表示门已开
- 手动关门后收到 `00`：表示门已关

### 关门反馈型锁

- 开锁后收到 `00`：表示门已开
- 手动关门后收到 `11`：表示门已关

## 7. 最简速查

开锁命令：

```text
8A [板地址] [锁地址] 11 [BCC]
```

开锁反馈：

```text
8A [板地址] [锁地址] [状态] [BCC]
```

手动关门后的主动反馈：

```text
81 [板地址] [锁地址] [状态] [BCC]
```

开启长通电：

```text
9A [板地址] [锁地址] 11 [BCC]
```

关闭长通电：

```text
9B [板地址] [锁地址] 11 [BCC]
```

一句话记忆：

- 开门反馈型锁：`11=开，00=关`
- 关门反馈型锁：`00=开，11=关`

## 8. 响应长度结论

不是“所有响应都同一个长度”，而是：

- 同一类响应通常是固定长度
- 不同类型响应，长度可能不同

目前这份 `demo` 里能确定的情况：

- 开锁反馈 `8A ...`：固定 `5` 字节
- 手动关门/微动主动上报 `81 ...`：固定 `5` 字节
- 电磁锁 `9A/9B` 的反馈：PDF 没明确列出专用格式，长度暂时不能完全写死

如果你后面要做串口解析，最稳妥的做法是：

1. 先按命令类型区分响应
2. 对 `8A` 和 `81` 按 `5` 字节解析
3. 对 `9A/9B` 先抓一帧实机返回，再把长度定死
